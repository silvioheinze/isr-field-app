{% trans "Introduction" %}

{% trans "ISR Field is a web application for managing spatial datasets: geometry points on a map, structured data entries per point, imports and exports, and collaboration with controlled access." %}

{% trans "After signing in, the Dashboard lists datasets you own, that are shared with you, or that are marked public. Use the top navigation to open Datasets or Typologies." %}

{% trans "Datasets" %}

  • {% trans "Create datasets from the dashboard or the dataset list (if your role allows creating datasets)." %}
  • {% trans "Each dataset has a detail page with shortcuts to settings, data input, entries, import/export, and more." %}
  • {% trans "Dataset settings let you rename the dataset, adjust options (e.g. multiple entries per geometry, mapping areas, anonymous data input), and configure default map position and zoom." %}
  • {% trans "You can copy a dataset to duplicate its structure and data according to the copy workflow." %}
  • {% trans "Public datasets are visible to all signed-in users; private datasets are only visible to the owner and invited users or groups." %}
  • {% trans "Clear data removes geometry and entries from a dataset—use with care." %}

{% trans "Sharing and access" %}

  • {% trans "The dataset access page lets the owner share the dataset with individual users and with groups." %}
  • {% trans "When mapping areas are used, you can restrict collaborators to specific mapping areas from this page (per user and per group)." %}
  • {% trans "Transfer ownership moves the dataset to another user; only the owner (or an administrator) can do this." %}

{% trans "Mapping areas" %}

{% trans "Enable mapping areas in dataset settings. The dataset owner (and administrators) can draw polygons on the data input map, name them, and allocate users to each area." %}

  • {% trans "Access limits can be set in two ways: on the dataset access screen (per user/group), or by allocating users on each mapping area in the mapping-areas panel." %}
  • {% trans "Collaborators restricted to a mapping area only see geometry points and entries that fall inside their allowed areas." %}
  • {% trans "Collaborators can see read-only outlines of their allowed areas on the map; they cannot select or edit those polygons." %}
  • {% trans "Only the dataset owner (and administrators) can open the full mapping-areas tools to draw, edit, or delete polygons." %}

{% trans "Fields and configuration" %}

  • {% trans "Field configuration defines which columns appear on data entry: labels, types (text, number, choice, typology-based fields, etc.), required flags, order, and help text." %}
  • {% trans "Custom fields add additional structured attributes beyond the standard set." %}
  • {% trans "Linking a field to a typology provides standardized dropdown choices from that typology." %}
  • {% trans "Non-editable fields can be shown but not changed during data entry." %}

{% trans "Data input" %}

  • {% trans "The data input view combines an interactive map with a panel to edit geometry details and yearly (or multiple) entries." %}
  • {% trans "Add points, focus the map, and use map controls (zoom, go to location, etc.) as provided on the page." %}
  • {% trans "If enabled for the dataset, multiple entries can exist per geometry (e.g. one row per year)." %}
  • {% trans "Anonymous data input allows contributors to use a special link without an account; those sessions are scoped to their own contributions." %}

{% trans "Import and export" %}

  • {% trans "CSV import walks you through column mapping and coordinate detection before data is written to the dataset." %}
  • {% trans "CSV export produces a spreadsheet with your field columns; options may include year-based layouts." %}
  • {% trans "File export can package attached files; large exports may use a background task and a download link when ready." %}

{% trans "Typologies" %}

  • {% trans "Typologies define reusable code lists (categories, codes, labels) for consistent classification." %}
  • {% trans "Create and edit typologies from the Typologies section; import or export typology entries from CSV when needed." %}
  • {% trans "Public typologies are visible to all users; private typologies are limited to their creator (and administrators)." %}

{% trans "Files and images" %}

  • {% trans "You can attach images and files to entries where the dataset supports it; downloads respect the same access rules as the entry and geometry." %}

{% trans "Users and administration" %}

  • {% trans "Staff and superuser roles control who can reach Django admin and who can manage users and groups in the application UI." %}
  • {% trans "User management (where available) lets administrators create users, reset passwords, and assign groups." %}
  • {% trans "Creating datasets and typologies may be limited to staff or administrators depending on site policy." %}

{% trans "Account and authentication" %}

  • {% trans "Sign in with your email and password; register for a new account if self-registration is enabled." %}
  • {% trans "Your profile page is where you can update email and change your password." %}
  • {% trans "Password reset uses email links if the feature is configured by the server administrator." %}
